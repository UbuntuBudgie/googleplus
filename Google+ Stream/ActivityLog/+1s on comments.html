+1'd comment on post by Ikey Doherty in Ubuntu Budgie+david mohammed this is excellent :) I understand re: the applet, my main goal was finding an area in which we can commonly work, and share resources, and make things better for everyone. Budgie has been in a bit of a disorganised lull for too long, so as much as 11 is about righting the wrongs in development, its also righting Budgie as a project. Budgie 11 should, in all ways, be more inclusive, and extended to the whole Budgie family. Budgie Remix is a huge part of that family, and indeed one of only 2 distros that push for Budgie to be better in a very public and dedicated way, by virtue of making it our primary desktop. It took even me a while to see some of the issues, and the fact is one of my primary complaints about other desktops in the past were very much tied to their parent project. I'm not comfortable being a hypocrite, which is why you see such a drive right now to enforce the independence of Budgie :) I look forward to working with you all in attaining that goal of making Budgie the #1 goto desktop, and honestly, us all enjoying it on an equal peer footing, is the most important part to me :) — The Eleventh Budgie As Budgie goes forward on the path to 11.0.0 - I'd like the Budgie Remix people to be a part of that. https://github.com/budgie-desktop/budgie-desktop/projects/1 We've seen all too often lately that a communication issue puts either the Budgie crew or the Remix crew into an odd light. I also dislike (as I'm sure others do) that there has to be any distancing at all, yet up until now it has been necessary. So, let's try to fix this. There are elements of Budgie Remix that are indeed very OS-specific. An example of this is the Budgie Welcome screen, it's tailor-made for Budgie Remix, and should obviously remain that way. An example of something that should likely be upstreamed, is some applets. One I would have liked to have seen some effort on upstreaming was the AppIndicator applet. It would appear that so far it is tied exclusively to Budgie Remix. So, here's what I'm proposing. As you all know, with Budgie 11, we've already moved Budgie Desktop out from underneath the Solus GitHub umbrella, into it's very own GitHub organisation. We also want to see more things happening in their own repos so that we're able to move away from our monolithic architecture. As part of "The Eleventh Budgie" workboard, issue 713 should be of particular interest: https://github.com/budgie-desktop/budgie-desktop/issues/713. All components within the Budgie Desktop GitHub organisation will lose their Solus branding, replaced with the wording: "<component_name> is a component of the Budgie Desktop", accompanied by the free Budgie Desktop logo. Previously this stated "is a Solus project", including the all-rights-reserved Solus project branding. This is us throwing the doors completely open here, as we drive home the absolute necessity for Budgie to be completely agnostic, and not tied to any one distro (i.e. Solus, or Remix, or anything). With that in mind, I would like to see that philosophy extended to downstreams, and I would like you to consider moving some of the currently Ubuntu specific components (such as the AppIndicator) into the new organisation, to facilitate first-class development in an agnostic fashion. I would far rather see Budgie Remix work together with the Budgie crew, than in silos. There is more than just Ubuntu and Solus in the world :) If you are interested, then let's start a discussion on the roadmap, and let me know what resources you need to accomplish this. I would like to see standardisation across all the repos. There is a "gotcha", of course. No official Budgie Desktop git repo should contain any packaging files of any kind. After all, distro agnostic, so no `debian` branches :) It should be noted I'm not just going for "distro-agnostic" here. Once the initial move to 11 is completed, Budgie itself will be portable enough, for once, to run on any *nix with the appropriate stack.
