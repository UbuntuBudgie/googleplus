Originally shared by SolusWe're currently evaluating some changes to Budgie's use of popovers, to fix some infamous issues (teleporting popovers, etc.)

At this point I can't promise that it lands, as it poses a few challenges. However, if it does, do note it will require an ABI and API break, and Budgie applets will require modification for the new version.

Long story short we're looking to finally nuke the GtkPopover usage in Budgie and replace them with a BudgiePopover type, which means we can remove lots and lots of hacks to support this, such as the popover placement being inconsistent, not being able to drag windows beyond the panel on multiple monitors, etc.

If I do this, and then break the API as a result, I'll throw some more goodies into the API to compensate. Apologies for having to take this route, but user experience does come first (And if you're an applet dev, your users will be happier as a result of the changes)
