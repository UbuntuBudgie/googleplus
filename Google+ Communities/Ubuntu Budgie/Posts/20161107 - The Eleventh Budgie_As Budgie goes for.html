The Eleventh Budgie

As Budgie goes forward on the path to 11.0.0 - I'd like the Budgie Remix people to be a part of that.

https://github.com/budgie-desktop/budgie-desktop/projects/1

We've seen all too often lately that a communication issue puts either the Budgie crew or the Remix crew into an odd light. I also dislike (as I'm sure others do) that there has to be any distancing at all, yet up until now it has been necessary.

So, let's try to fix this. There are elements of Budgie Remix that are indeed very OS-specific. An example of this is the Budgie Welcome screen, it's tailor-made for Budgie Remix, and should obviously remain that way.

An example of something that should likely be upstreamed, is some applets. One I would have liked to have seen some effort on upstreaming was the AppIndicator applet. It would appear that so far it is tied exclusively to Budgie Remix.

So, here's what I'm proposing. As you all know, with Budgie 11, we've already moved Budgie Desktop out from underneath the Solus GitHub umbrella, into it's very own GitHub organisation. We also want to see more things happening in their own repos so that we're able to move away from our monolithic architecture.

As part of "The Eleventh Budgie" workboard, issue 713 should be of particular interest: https://github.com/budgie-desktop/budgie-desktop/issues/713.

All components within the Budgie Desktop GitHub organisation will lose their Solus branding, replaced with the wording:

"<component_name> is a component of the Budgie Desktop", accompanied by the free Budgie Desktop logo. Previously this stated "is a Solus project", including the all-rights-reserved Solus project branding.

This is us throwing the doors completely open here, as we drive home the absolute necessity for Budgie to be completely agnostic, and not tied to any one distro (i.e. Solus, or Remix, or anything).

With that in mind, I would like to see that philosophy extended to downstreams, and I would like you to consider moving some of the currently Ubuntu specific components (such as the AppIndicator) into the new organisation, to facilitate first-class development in an agnostic fashion.

I would far rather see Budgie Remix work together with the Budgie crew, than in silos. There is more than just Ubuntu and Solus in the world :)

If you are interested, then let's start a discussion on the roadmap, and let me know what resources you need to accomplish this. I would like to see standardisation across all the repos. There is a "gotcha", of course. No official Budgie Desktop git repo should contain any packaging files of any kind. After all, distro agnostic, so no `debian` branches :)

It should be noted I'm not just going for "distro-agnostic" here. Once the initial move to 11 is completed, Budgie itself will be portable enough, for once, to run on any *nix with the appropriate stack.
